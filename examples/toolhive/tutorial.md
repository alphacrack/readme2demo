# Build ToolHive (thv) from Source

> ✅ Verified on 2026-07-03 · image readme2demo/base:latest · commit a935334

ToolHive is a lightweight, secure CLI written in Go for managing MCP (Model Context Protocol) servers using container isolation. In this quick tutorial, under a minute, you'll compile the `thv` CLI from source and confirm it runs correctly.

## Prerequisites

- Go 1.26 or newer
- Git

## Step 1 — Check your Go and Git versions

Before building anything, it's good practice to confirm the tools you need are installed and meet the version requirements. This command prints your installed Go and Git versions so you can verify your environment is ready.

```bash
go version && git --version
```

Expected output:

```text
go version go1.26.4 linux/arm64
git version 2.47.3
```

## Step 2 — Compile the thv CLI

This command compiles ToolHive's source code into a single executable binary. Go automatically downloads any missing dependencies listed in the project before compiling, then places the finished binary at `bin/thv` for you to run.

```bash
go build -o bin/thv ./cmd/thv
```

## Step 3 — See the available commands

Running the freshly built binary with no arguments prints its usage and help text, listing all the subcommands ToolHive offers. This is a quick way to confirm the build succeeded and to get a first look at what `thv` can do.

```bash
./bin/thv
```

Expected output:

```text
ToolHive (thv) is a lightweight, secure, and fast manager for MCP (Model Context Protocol) servers.
It is written in Go and has extensive test coverage—including input validation—to ensure reliability and security.

Under the hood, ToolHive acts as a very thin client for the Docker/Podman/Colima Unix socket API.
This design choice allows it to remain both efficient and lightweight while still providing powerful,
container-based isolation for running MCP servers.

Usage:

  thv [command]

  export      Export a workload's run configuration
```
